How this works. THRONDAR signs each tree head with ML-DSA-87 over the canonical core {v,model,answer_sha256,governance_flag,ts} under context throndar-sth-v1, where answer_sha256 = sha256(signed). A green verdict requires all of: the signature verifies against the pinned key (not the key the server sends), the digest chain holds, the context matches, and the receipt's key id matches the pin. Crypto bundle is served from this same origin. TestNet, unaudited.
